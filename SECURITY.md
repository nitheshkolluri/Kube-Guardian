# Security Policy

## Supported Versions

Only the latest major version of Kube-Guardian is currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 2.5.x   | :white_check_mark: |
| < 2.5.0 | :x:                |

## Reporting a Vulnerability

We take the security of our systems seriously. If you believe you have found a security vulnerability in Kube-Guardian, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

Please email us at **security@kubeguardian.io** with the following details:

1.  **Type of Vulnerability**: (e.g., XSS, Injection, Broken Auth)
2.  **Full Description**: Detailed steps to reproduce the issue.
3.  **Proof of Concept**: Code or screenshots demonstrating the vulnerability.

We will acknowledge your email within 48 hours and provide an estimated timeline for a fix.

### Policy

*   We ask that you do not disclose the vulnerability to the public until a patch has been released.
*   We will provide credit to the researcher who discovered the vulnerability (if desired) in the release notes.
